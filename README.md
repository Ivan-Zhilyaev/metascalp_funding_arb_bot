\## 🚀 Возможности



\- \*\*Автоматический поиск\*\* арбитражных возможностей на спотовом и фьючерсном рынках

\- \*\*Поддержка множества бирж\*\* через CCXT и MetaScalp API (14 бирж)

\- \*\*Два режима работы\*\*: Тестовый (симуляция) и Торговля (реальные сделки)

\- \*\*Веб-дашборд\*\* для мониторинга и управления настройками

\- \*\*Гибкие фильтры\*\*: объём торгов, спред, время до выплаты, статус торгов, комиссии

\- \*\*Управление рисками\*\*: проверка ликвидности, максимальное время удержания, смена знака ставки, защита от ликвидации

\- \*\*Авто-подключение к MetaScalp\*\* и синхронизация бирж

\- \*\*Telegram-бот\*\* для управления и уведомлений

\- \*\*Умное открытие позиций\*\*: последовательное открытие ног с откатом при неудаче

\- \*\*Защита от повторного открытия\*\*: блокировка символа на время исполнения

\- \*\*Логирование\*\* в файл и консоль

🎭 Режимы работы

Тестовый режим (test\_mode: true)

Симуляция сделок без отправки ордеров на биржу



Реалистичный расчёт PnL на основе:



Фактической ставки фандинга из CCXT



Реального времени удержания



Комиссий из настроек или CCXT



Безопасное тестирование стратегии



Режим Торговля (test\_mode: false)

Реальные ордера через MetaScalp API



Фактические комиссии из MetaScalp



Реальный PnL на основе истории фандинга из CCXT



Требует активного MetaScalp с настроенными биржами

🧠 Логика работы

Конечный автомат состояний

text

CLOSED → OPENING → OPENED → CLOSING → CLOSED

CLOSED: Ожидание сигнала



OPENING: Отправлены ордера на открытие



OPENED: Позиция открыта, мониторинг



CLOSING: Отправлены ордера на закрытие



Стратегия входа

Направление ставки	Действие на споте	Действие на перпе

Положительная (>0)	BUY SPOT	SHORT PERP

Отрицательная (<0)	SELL SPOT	LONG PERP

Алгоритм открытия (v2.6.2)

Проверка баланса через прямой запрос к MetaScalp



Открытие фьючерса первым



При успехе — открытие спота с запасом +1 токен



При неудаче спота — автоматический откат фьючерса



Блокировка символа от повторного открытия



Условия выхода

Смена знака ставки фандинга



Достижение максимального времени удержания (max\_position\_age\_hours)



Защита от ликвидации (max\_price\_change\_pct)



Ручное закрытие



Расчёт PnL

Тестовый режим:



text

avg\_rate = (entry\_rate + current\_rate) / 2

funding\_profit = size × |avg\_rate| × (holding\_hours / 8)

net\_pnl = funding\_profit - total\_fees

Режим Торговля:



text

net\_pnl = фактические\_выплаты\_фандинга - комиссии

❓ Часто задаваемые вопросы

1\. Бот не подключается к MetaScalp

Проверьте:



MetaScalp запущен



API-сервер включен в настройках MetaScalp



Порт в диапазоне 17845-17855 доступен



В WSL2 настроено зеркалирование сети



2\. Не появляются сигналы

Проверьте фильтры:



Увеличьте min\_profit\_bps или уменьшите пороги



Проверьте min\_volume\_24h\_usdt (возможно, слишком высокий)



Убедитесь, что active\_exchanges содержит нужные биржи



3\. Спот не открывается (ошибка баланса)

MetaScalp для spot принимает Size в USDT, не в токенах



Проверьте достаточность баланса с учётом комиссии



Бот автоматически добавляет запас +1 токен



4\. Как добавить новую биржу?

Добавьте биржу в enabled\_exchanges в config.yaml



В MetaScalp создайте подключения к этой бирже



Перезапустите бота



5\. Где посмотреть историю сделок?

В файле logs/trades.json и в дашборде на вкладке «История сделок».



6\. Как изменить настройки без перезапуска?

Через дашборд — все изменения применяются мгновенно (кроме добавления новых бирж).



7\. Как использовать Telegram-бота?

Создайте бота через @BotFather



Создайте файл .env с токенами



Включите telegram\_enabled: true в config.yaml



8\. Бот не закрывает позицию

Проверьте:



Достаточность баланса для закрытия



Не заблокирована ли позиция (close\_in\_progress)



Логи на наличие ошибок API
